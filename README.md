## Требования к MVP аналитического модуля маркетплейса

Финальный MVP аналитического модуля предназначен для демонстрации потоковой обработки пользовательских событий в маркетплейсе и расчёта ключевых метрик эффективности товаров в реальном времени. Модуль реализует концепцию, описанную в тезисе ВКР, обеспечивая полный конвейер: события> поток> агрегированная аналитика> визуализация для конечного пользователя.

### Ключевые функциональные требования:

1. Потоковая обработка событий:
    * Модуль подключается к существующему симулятору, который отправляет события пользователей через Kafka.
    * Обработка событий осуществляется в режиме реального времени с минимальной задержкой.
2. Хранение и агрегация данных:
    * Сырые события сохраняются в колоночной СУБД ClickHouse.
    * Агрегируются ключевые метрики: количество показов (globalCount), количество переходов (referCount), средняя позиция товаров (averageRank).
    * Агрегаты обновляются непрерывно по мере поступления событий.
3. Демонстрация визуальной динамики:
    * Простая веб-страница отображает изменение метрик по веб-сокету.
    * Визуализация служит подтверждением потокового характера обработки данных и полноты конвейера.
4. Гибкая архитектура и контейнеризация:
    * Модуль построен с использованием Hexagonal Architecture, отделяя бизнес-логику от инфраструктуры.
    * Все компоненты запускаются в Docker, что обеспечивает воспроизводимость среды и простоту тестирования.
5. Интерфейс и API:
    * REST API для получения агрегированных метрик.
    * Поддержка фильтрации по товарам, категориям и времени.

### Ожидаемый результат реализации
    
MVP демонстрирует прототип потокового аналитического модуля, который соответствует цели и задачам тезиса: от генерации событий пользователями до визуально наблюдаемой аналитики в реальном времени, с использованием колоночной базы и потоковой архитектуры. Продукт позволяет проверить гипотезы о поведении пользователей, продемонстрировать масштабируемость подхода и служит основой для дальнейшего расширения функционала в рамках бакалаврской работы.
